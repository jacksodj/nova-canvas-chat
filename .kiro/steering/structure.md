# Project Structure

## Repository Organization

```
bedrock-image-chat/
├── README.md                           # Main project documentation
├── Text to Image Bedrock Requirements.md  # Detailed requirements and design
├── LICENSE                             # Project license
├── .gitignore                          # Git ignore rules
├── docker-compose.yml                  # Local development environment
├── litellm-config.yaml                 # Model configuration for LiteLLM proxy
├── test-local.sh                       # Local testing automation script
├── docs/                               # Documentation
│   ├── FINE-TUNED-MODELS-GUIDE.md     # Quick start for custom models
│   └── LOCAL-TESTING-RESULTS.md        # Test validation report
└── infrastructure/                     # AWS deployment resources
    ├── README.md                       # Deployment guide
    ├── bedrock-image-chat-stack.yaml   # CloudFormation template
    ├── parameters.json                 # Deployment parameters
    ├── discover-models.sh              # Model discovery automation
    └── litellm-config-guide.md         # Advanced configuration guide
```

## Key Directories

### `/` (Root)
- **Configuration**: Core configuration files for local and cloud deployment
- **Documentation**: Main README and requirements documentation
- **Scripts**: Local testing and development automation

### `/docs/`
- **User Guides**: Step-by-step guides for common tasks
- **Testing**: Validation reports and test results
- **Focus**: End-user documentation and tutorials

### `/infrastructure/`
- **AWS Resources**: CloudFormation templates and deployment scripts
- **Configuration**: AWS-specific parameters and settings
- **Automation**: Model discovery and deployment tools
- **Focus**: Production deployment and operations

## File Naming Conventions

### Documentation
- `README.md` - Primary documentation for each directory
- `*-GUIDE.md` - Step-by-step user guides (ALL CAPS)
- `*-RESULTS.md` - Test reports and validation data (ALL CAPS)

### Configuration
- `*.yaml` - YAML configuration files
- `*.json` - JSON parameter files
- `docker-compose.yml` - Standard Docker Compose naming

### Scripts
- `*.sh` - Shell scripts with executable permissions
- `test-*.sh` - Testing automation scripts
- `discover-*.sh` - Discovery and automation tools

## Configuration File Hierarchy

1. **Local Development**: `docker-compose.yml` + `litellm-config.yaml`
2. **AWS Deployment**: `infrastructure/bedrock-image-chat-stack.yaml` + `infrastructure/parameters.json`
3. **Model Discovery**: `infrastructure/discover-models.sh` generates `litellm-config-generated.yaml`

## Working Directories

### For Local Development
```bash
# Work from project root
./test-local.sh
docker compose logs -f
```

### For AWS Deployment
```bash
# Work from infrastructure directory
cd infrastructure
./discover-models.sh us-east-1
aws cloudformation create-stack ...
```

### For Documentation Updates
```bash
# Documentation in docs/ directory
# Infrastructure docs in infrastructure/
# Main docs in project root
```

## Generated Files (Git Ignored)
- `litellm-config-generated.yaml` - Auto-generated by discovery script
- Docker volumes and container data
- AWS CloudFormation stack outputs
- Log files and temporary data

## Important File Relationships
- `litellm-config.yaml` ↔ `docker-compose.yml` (local development)
- `infrastructure/parameters.json` ↔ `infrastructure/bedrock-image-chat-stack.yaml` (AWS deployment)
- `infrastructure/discover-models.sh` → `litellm-config-generated.yaml` (model discovery)
- `docs/FINE-TUNED-MODELS-GUIDE.md` references `infrastructure/discover-models.sh`
